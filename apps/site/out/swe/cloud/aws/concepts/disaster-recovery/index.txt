14:Tdd5,<h1 id="disaster-recovery">Disaster Recovery</h1>
<ul>
<li>Qualquer evento que impacta negativamente a continuidade do negócio ou finanças de uma empresa</li>
<li><em>Disaster Recovery</em> se trata de preparar-se para se recuperar de um desastre</li>
</ul>
<h2 id="tipos">Tipos</h2>
<ul>
<li><em>On-premise</em> -> <em>On-premise</em>: Tradicional e custos</li>
<li><em>On-premise</em> -> <em>AWS Cloud</em>: Recuperação hibrida</li>
<li>AWS Cloud Region A -> AWS Cloud Region B</li>
</ul>
<h2 id="termos">Termos</h2>
<h3 id="recovery-point-objective-rpo">Recovery Point Objective (RPO)</h3>
<p>Quanto de perda de dados (RPO) é aceitável frente a um desastre, a perda de dados é calculada da data do último backup até a data do desastre</p>
<h3 id="recovery-time-objective-rto">Recovery Time Objective (RTO)</h3>
<p>Downtime (RTO) é calculado o tempo entre o desastre e a recuperação</p>
<h2 id="estratégias">Estratégias</h2>
<ul>
<li>Backup &#x26; Restore: Fácil, barato e com alto RPO e RTO</li>
<li>Pilot Light: Sistemas criticos estão executando sempre, apenas sistemas menos criticos devem ser recuperados</li>
<li>Warm Standby: Todos os componentes do sistema executando em uma versão com menos recursos que escala mediante a um desastre</li>
<li>Hot Site / Multi Site Approach: Todos os componentes do sistema rodando em produção com escala produtiva tanto no <em>on-premise</em> quanto na <em>AWS</em></li>
</ul>
<h2 id="dicas">Dicas</h2>
<h3 id="backup">Backup</h3>
<ul>
<li>Snapshots <em>EBS</em><a href="/swe/cloud/aws/services/ec2/ebs">ebs</a>, Backups automáticos <em>RDS</em><a href="/swe/cloud/aws/services/rds">rds</a>, <em>Snapshots</em> ...</li>
<li>Envios de dados regularmente ao S3, Glacier, ...</li>
<li>Para <em>On-premise</em>: <em>Snow Ball</em> <a href="/swe/cloud/aws/services/storage/snow-family">snow-family</a> ou <em>Storage Gateway</em> <a href="/swe/cloud/aws/services/storage/gateway">gateway</a></li>
</ul>
<h3 id="alta-disponibilidade">Alta disponibilidade</h3>
<ul>
<li><em>Route 53</em> <a href="/swe/cloud/aws/services/route53">route53</a> para migrar <em>DNS</em> <a href="/swe/network/concepts/dns">dns</a> de região para região</li>
<li><em>RDS Multi-AZ</em>, <em>Elasticache Multi-AZ</em>, EFS, S3</li>
<li><em>Site to Site VPN</em> <a href="/swe/cloud/aws/services/network/site-to-site-vpn">site-to-site-vpn</a> como recuperação do <em>Direct Connect</em> <a href="/swe/cloud/aws/services/network/direct-connect">direct-connect</a></li>
</ul>
<h3 id="replicação">Replicação</h3>
<ul>
<li>Replicação <em>RDS</em> (Cross Region) <a href="/swe/cloud/aws/services/rds">rds</a>, <em>AWS Aurora</em> <a href="/swe/cloud/aws/services/aurora">aurora</a></li>
<li>Replicação de dados <em>on-premise</em> para <em>RDS</em> <a href="/swe/cloud/aws/services/rds">rds</a></li>
</ul>
<h3 id="automação">Automação</h3>
<ul>
<li><em>CloudFormation</em>, <em>Elastic Beanstalk</em> para re-criar um ambiente completo</li>
<li>Recuperação, reinicialização de instâncias EC2 via alarmes <em>Cloud Watch</em> <a href="/swe/cloud/aws/services/monitoring-audit/cloud-watch">cloud-watch</a></li>
<li><em>Lambda functions</em> <a href="/swe/cloud/aws/services/serverless/lambda">lambda</a> para automações</li>
</ul>
<h3 id="caos">Caos</h3>
<ul>
<li>Ex: <em>Netflix</em> possui um exercito de "macacos" para derrubar maquinas EC2 em produção de maneira randômica</li>
</ul>
