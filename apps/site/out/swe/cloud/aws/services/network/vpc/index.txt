14:T1293,<h1 id="virtual-private-cloud-vpc">Virtual Private Cloud (VPC)</h1>
<ul>
<li>Todas novas contas AWS tem uma <strong>VPC padrão</strong></li>
<li>Novas instâncias EC2 são hospedadas na <em>VPC</em> padrão caso nenhuma subnets seja definida</li>
<li>A <em>VPC</em> padrão possui conectividade com a internet e todas instâncias EC2 hospedadas nela possuem endereços IPv4 públicos</li>
<li>É possível ter múltiplas VPCs em uma região da AWS (máximo de 5 VPCs por região)</li>
<li>É possível ter até 5 CIDR <a href="/swe/network/concepts/network-ipv4#CIDR">network-ipv4#CIDR</a> por VPC:
<ul>
<li>Mínimo <code>/28</code> (16 endereços de IP)</li>
<li>Máximo <code>/16</code> (65536 endereços de IP)</li>
</ul>
</li>
<li>Como a vpc é privada, apenas intervalos de IPv4 privados são permitidos:
<img src="/swe/network/concepts/network-ipv4#IPs-privados" alt="network-ipv4#IPs-privados"></li>
</ul>
<h2 id="resumo">Resumo</h2>
<ul>
<li>
<p><strong>CIDR – IP Range</strong>
Faixa de IPs usada para definir blocos de endereçamento.</p>
</li>
<li>
<p><strong>VPC – Virtual Private Cloud</strong>
Criamos uma lista de blocos CIDR IPv4 e IPv6 para definir a rede virtual.</p>
</li>
<li>
<p><strong>Subnets</strong>
Associadas a uma Availability Zone (AZ), cada uma possui um CIDR próprio.</p>
</li>
<li>
<p><strong>Internet Gateway</strong>
Configurado no nível da VPC, fornece acesso à Internet para IPv4 e IPv6.</p>
</li>
<li>
<p><strong>Route Tables</strong>
Precisam ser editadas para adicionar rotas de subnets para o Internet Gateway (IGW), VPC Peering Connections, VPC Endpoints, etc.</p>
</li>
<li>
<p><strong>Bastion Host</strong>
Instância EC2 pública usada para acesso SSH, com conectividade SSH para instâncias EC2 em subnets privadas.</p>
</li>
<li>
<p><strong>NAT Instances</strong>
Fornecem acesso à Internet para instâncias EC2 em subnets privadas. São antigas, devem ser configuradas em subnets públicas e ter o flag de verificação de origem/destino (Source/Destination Check) desativado.</p>
</li>
<li>
<p><strong>NAT Gateway</strong>
Serviço gerenciado pela AWS que fornece acesso escalável à Internet para instâncias EC2 privadas, quando o destino é um endereço IPv4.</p>
</li>
<li>
<p><strong>NACL – Network ACL</strong>
Regras stateless (sem estado) para tráfego de entrada e saída em nível de subnet. Não esqueça das portas efêmeras (Ephemeral Ports).</p>
</li>
<li>
<p><strong>Security Groups</strong>
Regras stateful (com estado), operam no nível da instância EC2.</p>
</li>
<li>
<p><strong>VPC Peering</strong>
Conecta duas VPCs com CIDRs que não se sobrepõem. Não é transitivo.</p>
</li>
<li>
<p><strong>VPC Endpoints</strong>
Fornecem acesso privado a serviços da AWS (S3, DynamoDB, CloudFormation, SSM) dentro de uma VPC.</p>
</li>
<li>
<p><strong>VPC Flow Logs</strong>
Podem ser configurados no nível da VPC, Subnet ou ENI para registrar tráfego ACCEPT e REJECT. Ajudam a identificar ataques e podem ser analisados com o Athena ou CloudWatch Logs Insights.</p>
</li>
<li>
<p><strong>Site-to-Site VPN</strong>
Configura um Customer Gateway no data center e um Virtual Private Gateway na VPC, estabelecendo uma VPN site-to-site pela Internet pública.</p>
</li>
<li>
<p><strong>AWS VPN CloudHub</strong>
Modelo de VPN em estrela (hub-and-spoke) para conectar múltiplos sites on-premise.</p>
</li>
<li>
<p><strong>Direct Connect</strong>
Configura um Virtual Private Gateway na VPC e estabelece uma conexão privada direta com um AWS Direct Connect Location.</p>
</li>
<li>
<p><strong>Direct Connect Gateway</strong>
Permite usar o Direct Connect com múltiplas VPCs em diferentes regiões da AWS.</p>
</li>
<li>
<p><strong>AWS PrivateLink / VPC Endpoint Services</strong></p>
<ul>
<li>Permite que clientes acessem serviços de forma privada da VPC do provedor de serviço.</li>
<li>Não requer VPC Peering, Internet pública, NAT Gateway ou Route Tables.</li>
<li>Deve ser usado com Network Load Balancer e ENI.</li>
</ul>
</li>
<li>
<p><strong>ClassicLink</strong>
Conecta instâncias EC2 no modo EC2-Classic à sua VPC de forma privada.</p>
</li>
<li>
<p><strong>Transit Gateway</strong>
Permite conexões de peering transitivas entre VPCs, VPNs e Direct Connect.</p>
</li>
<li>
<p><strong>Traffic Mirroring</strong>
Copia o tráfego de rede de ENIs para análise posterior.</p>
</li>
<li>
<p><strong>Egress-only Internet Gateway</strong>
Similar ao NAT Gateway, mas usado para fornecer saída para a Internet somente para IPv6.</p>
</li>
</ul>
