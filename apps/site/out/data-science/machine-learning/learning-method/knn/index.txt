<h2 id="o-que-é">O que é?</h2>
<blockquote>
<p>"K-Nearest Neighbors"</p>
</blockquote>
<p>Nós costumamos a julgar as pessoas pelo grupo ao qual elas se relacionam.</p>
<ul>
<li>Um método de aprendizagem supervisionada uma vez que sua variável alvo é conhecida</li>
<li>Algoritmo <em>Lazy</em> (Não há fase de treinamento, apenas armazenamento dos dados), Logo ele não aprende, mas memoriza</li>
<li>Cálculo de distância entre os dados para realizar a classificação (similaridade entre features)</li>
</ul>
<h3 id="características">Características</h3>
<ul>
<li>Util para pequenas bases de dados</li>
<li>Multi classe e binária</li>
<li>Baseado em instâncias (Instance-based learning)</li>
</ul>
<h2 id="complexidade">Complexidade</h2>
<h2 id="hiperparâmetro-k">Hiperparâmetro K</h2>
<ul>
<li>K comumente é um número ímpar para evitar empates em classificação binária</li>
<li>Quanto menor o K, maior o risco de <em>Overfitting</em> <a href="/data-science/machine-learning/glossary/overfitting">overfitting</a></li>
<li>Quanto maior o K, maior o risco de <em>Underfitting</em></li>
<li>K pode ser ajustado com validação cruzada</li>
<li>K tem uma relação inversa com variância, uma vez que o K aumenta a variância diminui</li>
</ul>
<h3 id="como-escolher-o-valor-de-k">Como escolher o valor de K?</h3>
<ul>
<li>Plotar a <em>taxa de acurácia</em> ou <em>F1 Score</em> para diferentes valores de K</li>
</ul>
<h2 id="etapas">Etapas</h2>
<ol>
<li>Calcule a <a href="#dist%C3%A2ncias">distância</a> entre o novo objeto sem classificação e todos objetos classificados existentes, estas distâncias serão armazenadas em memória</li>
<li>Ordene de forma ascendente as distâncias calculadas</li>
<li>Armazene os <em>K</em> pontos mais próximos</li>
<li>Calcule a proporção de cada classe e armazene em memória</li>
<li>Atribua a classe com maior proporção</li>
</ol>
<h3 id="distâncias">Distâncias</h3>
<h4 id="euclidiana-l2-norm">Euclidiana (L2 norm)</h4>
<p>$$
d(p,q) = \sqrt{\sum_{i=1}^{n} (p_i - q_i)^2}
$$</p>

<h4 id="jaccard">Jaccard</h4>
<h4 id="hamming">Hamming</h4>
<ul>
<li>Utilizada para valores categóricos, nos diz se dois valores categóricos são iguais.</li>
</ul>
<h2 id="prós">Prós</h2>
<ul>
<li>Algoritmo simples, fácil de se interpretar</li>
<li>Não paramétrico (não faz suposições sobre a distribuição dos dados)</li>
<li>Utilizado para classificação e regressão</li>
<li>Treinamento rápido comparado a outros algoritmos</li>
</ul>
<h2 id="contras">Contras</h2>
<ul>
<li>Não possui uma fase de treino, então o custo computacional pode ser alto na fase de predição. Toda vez que uma predição é feita, o algoritmo precisa calcular a distância entre o ponto a ser classificado e todos os outros pontos do conjunto de dados de treinamento. Isso pode ser melhorado com estruturas de dados como <em>KD-Trees</em> ou <em>Ball-Trees</em>, mas ainda sim pode ser inssuficiente para conjuntos de dados muito grandes.</li>
<li>Alto requerimento de memória uma vez que o KNN precisa armazenar todos pontos de dados</li>
<li>A eficiência do algoritmo diminui a medida que o conjunto de dados aumenta</li>
<li>O algoritmo não lida bem com <em>missings</em>, sendo necessário preencher os dados faltantes</li>
<li>Sofre de distribuição desigual de classes (classes majoritárias tendem a dominar a classificação)</li>
<li>A acurácia é deteriorada em dados com alta dimensionalidade (muitas features). KNN sofre com a "maldição da dimensionalidade", onde a distância entre os pontos se torna menos significativas à medida que o número de dimensões aumenta. Isso pode levar a classificações imprecisas.</li>
<li>Sensível a outliers, a acurácia é comprometida por dados irrelevantes.</li>
</ul>
<h2 id="aplicação">Aplicação</h2>
<ul>
<li>Sistemas de recomendação para anúncios no Youtube ou produtos na Amazon.</li>
<li>Utilização no varejo para detectar padrões em uso de cartões de crédito e identificar possíveis fraudes</li>
<li>Predição de votos potências em eleições com base em dados demográficos e históricos de votação</li>
<li>Reconhecimento de video</li>
<li>Reconhecimento de imagem</li>
<li>Reconhecimento de escrita de mão</li>
</ul>
<h2 id="comparação">Comparação</h2>
<ul>
<li><strong>KNN</strong> é um algoritmo de machine learning supervisionado, enquanto <strong>K-means</strong> é um algoritmo não supervisionado</li>
<li><strong>KNN</strong> é utilizado para classificação assim como regressão e <strong>K-means</strong> para clusterização</li>
<li><em>K</em> in <strong>KNN</strong> é o número de vizinhos mais próximos e o <em>K</em> em <strong>K-means</strong> é o número de clusters que estamos tentando identificar nos dados.</li>
</ul>
<h2 id="dicas">Dicas</h2>
<ul>
<li>Comece pelo KNN quando o tipo de distribuição não é conhecida</li>
<li>Para evitar empate entre as classes escolha um valor de <code>K</code> impar</li>
<li>Escolha um K pequeno</li>
<li>o Knn não gera uma função de saída, então uma alternativa é calcular centroids para cada classe, gerando um objeto único para cada classe. Logo o K será igual a 1 e o que estiver mais próximo definirá a classe</li>
<li>O KNN pode ser implementado utilizando somente SQL e pode se beneficiar da indexação do banco de dados</li>
<li>Utilize <em>Weighted voting</em></li>
<li>Features precisam ter a mesma escala, normalize ou padronize os dados</li>
<li>Reduza a dimensionalidade via <em>PCA (Principal Component Analysis)</em></li>
<li>Aproxime os vizinhos mais próximos utilizando <em>k-d trees</em> para armazenar observações de treino e reduzir o tempo de teste quando houver menos de 20 dimensões</li>
<li>Caso a dimensionalidade seja muito alta utilize LHS (sensitive hashing)</li>
<li>Utilize K-means para reduzir o número de pontos utilizados para computar a distância</li>
<li>Utilize uma amostra aleatória para reduzir o conjunto de dados</li>
</ul>
