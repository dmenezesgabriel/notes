15:T1124,<h1 id="wiki">Wiki</h1>
<p>This file describes the conventions and workflows the LLM agent should follow when ingesting sources, updating the wiki, and answering queries. Keep it short and prescriptive.</p>
<h2 id="frontmatter-conventions-yaml">Frontmatter conventions (YAML)</h2>
<ul>
<li>title: Human-friendly title</li>
<li>aliases: list of alternative short titles (optional)</li>
<li>desc: short description/synopsis</li>
<li>tags: list of tags (optional)</li>
<li>sources: list of source ids/paths used to build this page (optional)</li>
<li>lastUpdated: ISO timestamp</li>
<li>pageType: one of [entity, concept, index, note, log]</li>
<li>status: draft | verified | flagged</li>
<li>zettelId: optional unique id (when pageType: permanent) — format ZK-YYYYMMDDHHMMSS-XXX</li>
</ul>
<h3 id="zettelkasten">Zettelkasten</h3>
<ul>
<li>pageType: extend values to include [fleeting, literature, permanent]</li>
<li>zettelId: unique Zettelkasten id for evergreen notes (e.g. ZK-20260425183000-001)</li>
<li>Use pageType: literature for source summaries and include a <code>sources:</code> list in frontmatter.</li>
<li>Use pageType: permanent for atomic evergreen notes; include zettelId and link to related notes explicitly.</li>
<li>Keep fleeting notes short and ephemeral; review them and convert to literature/permanent as needed.</li>
</ul>
<h2 id="file-types">File types</h2>
<ul>
<li>index.md (domain-level): catalog of pages for a domain. Updated on each ingest.</li>
<li>log.md: append-only chronological log of ingest/query/lint actions.</li>
<li>entity pages: pages that represent things (people, services, products) and aggregate references.</li>
<li>concept pages: explainers, comparisons, definitions.</li>
</ul>
<h2 id="ingest-workflow-prescriptive">Ingest workflow (prescriptive)</h2>
<ol>
<li>Read full source S.</li>
<li>Generate a 1-paragraph summary and a list of named entities/concepts mentioned.</li>
<li>For each relevant entity/concept:
<ul>
<li>If page exists: update page, append a one-line note to sources and add changed date.</li>
<li>If not: create a stub page with frontmatter, a short summary, and links back to the source.</li>
</ul>
</li>
<li>Update domain index.md to include/refresh links to changed pages.</li>
<li>Append an entry to log.md: e.g. "## [2026-04-24] ingest | <source title=""> — created 3 pages, updated 5 pages"</li>
<li>Flag contradictions: if new source conflicts with previous claims, add a "Conflicts" section to affected pages and add a short explanation + source pointers.</li>
</ol>
<h2 id="query-workflow">Query workflow</h2>
<ul>
<li>Search index.md first to shortlist pages.</li>
<li>Use search to retrieve candidate pages and snippets.</li>
<li>Synthesize answer with explicit citations (page path + short excerpt).</li>
<li>When the answer is substantial, offer to file it as a new wiki page ("Would you like me to save this answer as a page?"). If yes, create page with frontmatter listing contributing sources.</li>
</ul>
<h2 id="lint-workflow-periodic-or-on-demand">Lint workflow (periodic or on-demand)</h2>
<ul>
<li>Detect orphan pages (no incoming links) and propose parent pages or index placement.</li>
<li>Detect duplicate titles and propose aliases or disambiguated titles.</li>
<li>Detect missing cross-references for frequent terms without pages.</li>
<li>Detect contradictions across pages and open a "conflicts" ticket in log.md.</li>
</ul>
<h2 id="safety-guardrails">Safety guardrails</h2>
<ul>
<li>Never assert factual claims without a source; prefer phrasing "According to [source], ..." for things not corroborated by >=2 sources.</li>
<li>When unsure, add "needs verification" to page status and add the investigative query to the log.</li>
<li>Keep the human in-the-loop for destructive edits: major renames, merges, or deletions require explicit human confirmation.</li>
</ul>
<h2 id="operational-notes">Operational notes</h2>
<ul>
<li>Index.md and log.md are the main control surfaces for the agent.</li>
<li>Keep a short list of canonical tag pages (notes/tags.<tag>.md) for widely used tags.</tag></li>
</ul>

<li>updated: data-science/concept-y</li>
<li>notes: flagged contradiction between concept-y and concept-z (see links)</li>
